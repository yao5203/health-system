# 数据库表清单

本项目使用 MySQL 保存用户、医生、健康评测、康养方案、每日打卡、咨询协作和后台维护等数据。数据库不能像普通文件一样直接上传到 GitHub，因此仓库中提供 `schema.sql` 作为建库和初始化脚本。

## 基础角色表

| 表名 | 中文含义 | 主要用途 |
| --- | --- | --- |
| `user` | 普通用户表 | 保存普通用户账号、密码、年龄、性别、手机号等基础信息。 |
| `doctor` | 医生表 | 保存医生账号、专业方向、擅长标签、简介和账号状态。 |
| `system_admin` | 系统管理员表 | 保存管理员登录账号、显示名称和账号状态。 |

## 健康评测表

| 表名 | 中文含义 | 主要用途 |
| --- | --- | --- |
| `health_data` | 健康指标表 | 保存用户录入的血压、血糖、身高、体重、心率、血氧等硬性指标。 |
| `question` | 问卷题目表 | 保存问卷题目内容、题目类型、体质分类、评测维度、适用体质、排序和启用状态。 |
| `question_option` | 问卷选项表 | 保存每道题目的选项内容和选项分值。 |
| `user_answer` | 用户问卷作答表 | 保存用户提交的题目编号、选项编号和作答时间。 |
| `health_result` | 健康评测结果表 | 保存综合得分、健康等级、体质倾向和生成时间。 |
| `questionnaire_quality` | 问卷质量表 | 保存答题时长、答题数量、漏答数量、直线作答比例和可信度提示。 |

## 康养服务表

| 表名 | 中文含义 | 主要用途 |
| --- | --- | --- |
| `health_plan` | 康养方案表 | 保存饮食、饮水、运动和生活方式建议。 |
| `health_plan_template` | 康养方案模板表 | 保存更细粒度的推荐模板，用于按体质、风险和维度短板组合建议。 |
| `plan_favorite` | 方案收藏表 | 保存用户收藏过的康养方案。 |
| `daily_checkin` | 每日打卡表 | 保存睡眠时长、运动时长、压力水平、心情得分、体重和备注。 |
| `health_article` | 健康资讯表 | 保存健康文章标题、分类、摘要、正文、标签和发布状态。 |

## 咨询协作表

| 表名 | 中文含义 | 主要用途 |
| --- | --- | --- |
| `consultation_request` | 咨询申请表 | 保存用户发起的咨询主题、问题详情、偏好标签、分配医生和处理状态。 |
| `consultation_message` | 咨询消息表 | 保存用户和医生在咨询会话中的消息内容。 |
| `change_proposal` | 医生修改提案表 | 保存医生针对题库、资讯、方案等内容提交的修改建议，由管理员审核。 |

## 后台维护表

| 表名 | 中文含义 | 主要用途 |
| --- | --- | --- |
| `rule_config` | 评测规则配置表 | 保存健康等级、风险等级等阈值配置，便于后台调整规则。 |
| `feedback` | 用户反馈表 | 保存普通用户提交的意见反馈。 |
| `system_log` | 系统日志表 | 保存管理员新增、修改、删除等后台操作记录。 |
